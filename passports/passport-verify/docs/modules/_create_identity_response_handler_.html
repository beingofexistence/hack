create-identity-response-handler

Helper function to make handling callbacks from passport.authenticate('verify', callback) easier.

This is necessary because there are a number of different scenarios that services need to handle when dealing with Verify responses. passport.js isn't aware of these, so the interface it provides is not ideal.

    The createIdentityResponseHandler function should be used if your service connects to GOV.UK Verify using the Verify Service Provider (VSP) without a Matching Service Adapter (MSA)

    This function makes it easier to handle the passport.authenticate() callback. The function takes separate callbacks for each of the response scenarios you have to handle and returns a function that will invoke the appropriate callback when called by passport.

    Parameters

    • responseScenarios: IdentityResponseScenarios

      Callbacks to handle each type of response that GOV.UK Verify can return
