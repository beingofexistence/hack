createIdentityStrategy

  • createIdentityStrategy(verifyServiceProviderHost: string, handleIdentity: function, saveRequestId: function, loadRequestId: function, serviceEntityId?: undefined | string, samlFormTemplateName?: undefined | string, levelOfAssurance?: undefined | "LEVEL_1" | "LEVEL_2"): PassportVerifyStrategy
    • Defined in passport-verify-strategy.ts:224

    Creates an instance of PassportVerifyStrategy

    This version of the function should only be used if your service connects to GOV.UK Verify using the Verify Service Provider (VSP) without a Matching Service Adapter (MSA).

    Parameters

    • verifyServiceProviderHost: string

      The URL that the Verify Service Provider is running on (e.g. http://localhost:50400)

    • handleIdentity: function

      A callback that will be invoked when a response with an identity is received. The identity object will contain the users' attributes (i.e. firstName, surname etc.). Your callback should store details of the user in your datastore and return an object representing the user.

        • (identity: TranslatedIdentityResponseBody): object | false

        • Parameters

          • identity: TranslatedIdentityResponseBody

          Returns object | false

    • saveRequestId: function

      A callback that will be invoked to save the requestId that has been generated by the verify service provider. Your callback should save the request Id in a secure manner so that it can be matched against the corresponding SAML response.

        • (requestId: string, request: express.Request): void

        • Parameters

          • requestId: string
          • request: express.Request

          Returns void

    • loadRequestId: function

      A callback that will be invoked to load the requestId that has been securely saved for the user's session.

        • (request: express.Request): string

        • Parameters

          • request: express.Request

          Returns string

    • Optional serviceEntityId: undefined | string

      (Optional) The entityId that will be passed to the Verify Service Provider. This is only required if the service provider is configured to be multi tenanted.

    • Optional samlFormTemplateName: undefined | string

      (Optional) The name of a template in your service which will provide the form used to post an authn request. If present, this will be rendered with the ssoLocation and samlRequest passed in. Otherwise, a default form will be used. You should use this option if you wish to style the form, which should be autoposting so only seen if the user has javascript disabled, to match the rest of your service.

    • Optional levelOfAssurance: undefined | "LEVEL_1" | "LEVEL_2"

      (Optional) LEVEL_1 or LEVEL_2 - defaults to LEVEL_2. The Level of Assurance to request from the Verify Service Provider and the minimum level to expect in the Response (e.g. if you specify LEVEL_1 a LEVEL_2 Response would also be permissible).

    Returns PassportVerifyStrategy

    A strategy to be registered in passport with

    passport.use(passportVerifyStrategy)
