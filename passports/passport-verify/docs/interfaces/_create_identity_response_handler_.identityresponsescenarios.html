Interface IdentityResponseScenarios

The callbacks in the IdentityResponseScenarios interface define scenarios that can occur when handling a response from GOV.UK Verify.

Services should handle each of these scenarios appropriately.

onAuthnFailed

onAuthnFailed: function
  • Defined in create-identity-response-handler.ts:45

Called when the user failed to verify, for example if the user got their password wrong.

Your callback should redirect the user to a page offering them other ways to use your service (e.g. using a non-verify way of proving their identity or going somewhere in person).

onError

onError: function
  • Defined in create-identity-response-handler.ts:69

Called when the response from verify can't be handled correctly (for example if its signature is invalid or if its validUntil date is in the past), or if the response from verify represents an error.

Your callback should make sure the error is logged appropriately and render an error page telling the user that there are technical problems with Verify.

The callback will be provided with an error object, which will provide details of what went wrong.

onIdentityVerified

onIdentityVerified: function
  • Defined in create-identity-response-handler.ts:35

Called when handling an IDENTITY_VERIFIED response. Your callback should redirect the user to the appropriate page so they can begin using your service.

The callback will be passed an identity object, which will be the output from your handleIdentity callback.

onNoAuthentication

onNoAuthentication: function
  • Defined in create-identity-response-handler.ts:55

Called when the user fails to authenticate or times out at the identity provider (e.g. because they don't have the required documentation with them).

Your callback should redirect the user to a page offering them other ways to use your service (e.g. using a non-verify way of proving their identity or going somewhere in person).
